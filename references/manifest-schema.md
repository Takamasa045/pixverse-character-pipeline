# Manifest Schema

The pipeline uses two manifest formats depending on the mode of operation.

## Creative Workflow Manifest (Mode A)

Hand-crafted `manifest.json` used by Remotion's CharacterVideo component. Agent builds this during Phase 5.

```json
{
  "bgm": "assets/audio/bgm.mp3",
  "bgmVolume": 0.12,
  "speakerImage": "assets/speaker.jpg",
  "theme": {
    "background": "#1a2e1a",
    "accent": "#D4913A",
    "text": "#ffffff"
  },
  "cuts": [
    {
      "id": "hook",
      "startFrame": 0,
      "durationInFrames": 120,
      "videoSrc": "assets/cuts/cut-hook.mp4",
      "imageSrc": null,
      "overlayText": "",
      "overlayStyle": "none",
      "hasAudio": false
    },
    {
      "id": "talking",
      "startFrame": 120,
      "durationInFrames": 150,
      "videoSrc": "assets/cuts/cut-talking.mp4",
      "imageSrc": null,
      "overlayText": "字幕テキスト",
      "overlayStyle": "subtitle",
      "hasAudio": true
    }
  ]
}
```

### Fields

| Field | Type | Description |
|-------|------|-------------|
| `bgm` | `string\|null` | BGM audio file path (relative to `public/`) |
| `bgmVolume` | `number` | BGM volume (0.0 - 1.0) |
| `speakerImage` | `string\|null` | Speaker thumbnail image |
| `theme` | `object` | `{ background, accent, text }` hex colors |
| `cuts[].id` | `string` | Unique clip identifier |
| `cuts[].startFrame` | `number` | Frame offset (cumulative) |
| `cuts[].durationInFrames` | `number` | Clip length in frames |
| `cuts[].videoSrc` | `string\|null` | Video file path |
| `cuts[].imageSrc` | `string\|null` | Image file path (for endcards) |
| `cuts[].overlayText` | `string` | Text overlay content |
| `cuts[].overlayStyle` | `string` | `"none"` / `"subtitle"` / `"title"` / `"lower-third"` / `"endcard"` |
| `cuts[].hasAudio` | `boolean` | Use the clip's audio track |

## Pipeline Render Manifest (Mode B)

Auto-generated `manifest.render.json` per `(language, aspect_ratio)`. Used by Remotion to render the final MP4.

```json
{
  "projectSlug": "mixed-generated",
  "language": "ja",
  "aspectRatio": "16:9",
  "fps": 30,
  "width": 1920,
  "height": 1080,
  "durationInFrames": 165,
  "speakerImage": ".pipeline/mixed-generated/20260321T073752Z/ja/16x9/assets/speaker.svg",
  "bgm": null,
  "bgmVolume": 0.15,
  "theme": {
    "background": "#111827",
    "accent": "#f97316",
    "text": "#ffffff"
  },
  "cuts": [
    {
      "id": "opener",
      "startFrame": 0,
      "durationInFrames": 60,
      "videoSrc": ".pipeline/mixed-generated/20260321T073752Z/ja/16x9/assets/opener.mp4",
      "imageSrc": null,
      "overlayText": "Spring Launch",
      "overlayStyle": "title",
      "hasAudio": true
    }
  ]
}
```

## Notes

- `clipAssets` in the run manifest stores output-relative paths for auditing.
- `speakerImage`, `bgm`, `videoSrc`, `imageSrc` in render manifests are `public/`-relative paths used by Remotion.
- `planned` status appears when `pipeline:run --dry-run` is used.
- Mode A manifests are hand-crafted; Mode B manifests are auto-generated by `./bin/pipeline run`.
